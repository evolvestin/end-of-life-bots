# End-of-Life Bots Infrastructure

**Дата инициализации проекта:** Ноябрь 2025 г.

## Описание

Данный программный комплекс (`end-of-life-bots`) представляет собой специализированное архитектурное решение, разработанное для централизованного управления процессом вывода из эксплуатации (decommissioning) пула Telegram-ботов.

Основная задача системы — обеспечение корректного завершения жизненного цикла программных продуктов, информирование конечных пользователей о прекращении поддержки и сохранение аудиторского следа остаточной активности.

## Функциональное назначение

Система работает в режиме "заглушки" (stub) для списка токенов, указанных в конфигурации, и выполняет следующие функции:

1.  **Автоматическое уведомление пользователей:**
    При любой входящей активности (сообщение, команда, добавление в чат) бот отправляет стандартизированное уведомление (`END_OF_LIFE_MESSAGE`) о полном прекращении обслуживания, остановке серверов и удалении баз данных.

2.  **Агрегация и логирование событий:**
    Несмотря на прекращение обслуживания, система продолжает вести детальный мониторинг входящего трафика.
    *   Все входящие сообщения, сервисные уведомления и изменения статусов участников логируются в специальный административный канал.
    *   Поддерживается HTML-форматирование логов с сохранением структуры оригинальных сообщений.

3.  **Архивация медиа-контента:**
    Входящие медиа-файлы (фото, документы, голосовые сообщения) обрабатываются и пересылаются в отдельный канал-хранилище (`ID_MEDIA`) для целей возможного пост-анализа или архивации.

4.  **Мультибот-архитектура:**
    Одно приложение (`main.py`) способно обслуживать неограниченное количество токенов одновременно, что позволяет экономить серверные ресурсы при поддержке "кладбища" неактивных проектов.

## Технический стек

*   **Язык:** Python 3.12+
*   **Фреймворк:** aiogram 3.x
*   **Контейнеризация:** Docker / Docker Compose

## Установка и запуск

1.  Клонируйте репозиторий.
2.  Создайте файл `.env` на основе примера, заполнив переменные:
    *   `TOKENS`: Список токенов ботов через запятую.
    *   `DEV_TOKEN`: Токен сервисного бота для отчетов об ошибках.
    *   `ID_DEV`, `ID_LOGS`, `ID_MEDIA`, `ID_FORWARD`: ID целевых чатов/каналов для логов.
    *   `LINK_TO_ADMIN`: Контакт для обратной связи в сообщении о закрытии.
3.  Запустите проект через Docker Compose:

```bash
docker-compose up -d --build
```
